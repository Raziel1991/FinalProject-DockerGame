# Docker Heist Presentation Narrative Plan

Audience: COMP-308 instructor and class during final group project presentation.

Objective: Present Docker Heist as a completed Exercise 2 web-based game: React Vite + Three.js frontend, Express/Apollo GraphQL backend, MongoDB persistence, JWT auth, gameplay, rewards, challenge mode, and testing.

Narrative Arc:
1. Introduce the game concept and value.
2. Show the core gameplay loop.
3. Explain the frontend/backend/database architecture.
4. Demonstrate GraphQL and MongoDB responsibilities.
5. Highlight rewards, AI challenge mode, and live events.
6. Map implementation to the assignment rubric.
7. Close with testing, limitations, and demo steps.

Slide List:
1. Title / project overview.
2. Assignment target and completed scope.
3. Gameplay loop.
4. System architecture.
5. Frontend implementation.
6. Backend GraphQL API.
7. MongoDB data model.
8. Rewards, AI challenge, and live events.
9. Rubric coverage.
10. Testing, demo flow, and next steps.

Visual System:
Dark tech/company aesthetic inspired by the updated UI: charcoal background, cyan/green/amber accents, terminal motifs, glass panels, clean diagrams, 8px radius, and modern sans typography.

Editability Plan:
All slide text, tables, diagrams, labels, and cards will be editable PowerPoint objects. Decorative visuals will be built from simple editable shapes.
